# Filestore

一个无需提交者登录的轻量级文件收集管理系统。

Filestore 面向班级作业、考试材料、毕业材料、证件照片等场景。管理员创建收集任务后，系统生成随机提交链接或二维码；提交者通过链接填写表单并上传文件，无需注册登录。

## 核心流程

```text
管理员创建收集任务
↓
配置表单字段、文件规则、命名规则、截止时间
↓
生成提交链接或二维码
↓
提交者填写信息并上传文件
↓
系统校验并自动整理文件
↓
管理员查看、筛选、导出、下载
```

## 主要功能

- 无提交者账号：提交者通过随机链接或二维码进入提交页。
- 管理员登录：使用管理员密码进入后台，支持在系统设置中修改密码。
- 自定义任务：支持任务标题、说明、截止时间、开放/停止提交状态。
- 自定义表单：可配置姓名、学号、考试号、班级、邮箱等字段。
- 正则校验：字段可配置正则规则，例如中文姓名、固定学号、固定考试号。
- 默认模板：内置学号模板、考试号模板，支持保存当前任务为自定义模板。
- 文件上传规则：限制文件类型、单文件大小、上传数量。
- 提交端文件管理：上传前可视化展示文件，图片显示缩略图，支持拖拽排序、上移、下移、删除。
- 重命名预览：提交者可在上传前看到文件将被保存成什么名字。
- 重复提交覆盖：相同姓名和学号/考试号再次提交会覆盖旧提交，适合提交错误后重新上传。
- 文件自动整理：支持按字段自动命名文件，并支持取字段前几位、后几位。
- 文件夹命名规则：多文件提交时可按规则为每个提交者创建文件夹。
- 提交管理：查看提交记录、搜索、删除错误提交。
- 文件管理：查看文件、下载单个文件、删除文件。
- 缺交统计：可配置应提交名单，系统自动显示缺交名单并支持复制。
- 导出 CSV：导出提交汇总表。
- 浏览器端 ZIP 打包：批量下载时由管理员浏览器打包，服务器不再生成 ZIP。
- 站点个性化：支持设置站点标题、站点地址，适合反向代理到域名。
- 移动端适配：提交页和管理页均针对手机屏幕做了响应式优化。
- 全局版权页脚：页面底部展示项目 GitHub 地址。

## 命名规则

文件命名和文件夹命名是两套独立规则。

常用变量：

```text
{name}          姓名
{student_id}    学号或考试号
{original}      原文件名
{index}         多文件序号
```

支持截取字段：

```text
{student_id|last:2}   取学号/考试号后 2 位
{student_id|first:4}  取学号/考试号前 4 位
```

示例：

```text
文件命名规则：{name}-{student_id}
文件夹命名规则：{name}-{student_id|last:2}
```

单文件提交时：

```text
张三-2420150508.pdf
```

多文件提交时：

```text
张三-08/
  张三-2420150508-1.jpg
  张三-2420150508-2.jpg
```

如果命名规则中包含 `{index}`，单文件提交不会额外显示 `1`；多文件提交才会显示序号。

## 缺交名单

应提交名单用于缺交统计，一行一个值。推荐填写学号或考试号：

```text
2020240444
2020240445
2020240446
```

系统会优先用提交记录中的学号/考试号匹配；没有编号字段时再使用姓名匹配。

## 启动

```powershell
python app.py
```

默认访问地址：

```text
http://127.0.0.1:8964
```

默认管理员密码：

```text
admin123
```

可通过环境变量修改默认端口：

```powershell
$env:PORT="8964"
python app.py
```

可通过环境变量修改初始管理员密码：

```powershell
$env:FILESTORE_ADMIN_PASSWORD="your-strong-password"
python app.py
```

管理员密码也可以登录后在 `系统设置` 中修改。

## 数据目录

默认数据位置：

```text
data/filestore.db
uploads/
```

- `data/filestore.db`：SQLite 数据库
- `uploads/`：上传文件

## 反向代理

如果需要通过域名访问，登录管理端后进入 `系统设置`，填写站点地址，例如：

```text
https://files.example.com
```

保存后，提交链接和二维码会使用该对外地址。留空时使用当前访问地址。

## 批量下载

批量 ZIP 下载由浏览器端完成：

- 服务器只提供单文件下载。
- 管理员浏览器逐个读取文件并生成 ZIP。
- 多文件提交者会在 ZIP 中生成个人文件夹。
- 单文件提交者直接放在 ZIP 根目录。

ZIP 目前使用无压缩打包模式。图片、PDF、docx 通常已经压缩过，这样可以减少服务器压力并提升稳定性。
